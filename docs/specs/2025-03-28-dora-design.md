# Dora - AI 情感陪伴桌面应用设计文档

**版本**: 1.0
**日期**: 2025-03-28
**状态**: 设计阶段

---

## 1. 项目概述

### 1.1 产品定位
Dora 是一款 macOS 平台的 AI 情感陪伴桌面应用，以哆啦A梦为灵感，打造一个温暖、智能、有用的桌面伙伴。

### 1.2 核心目标
- **情感陪伴**: 像真正的朋友一样倾听、理解、回应
- **实用工具**: 提供便捷的桌面小工具（便签、倒计时等）
- **低干扰**: 轻量运行，不打扰用户工作流
- **隐私优先**: 本地优先，数据可控

### 1.3 目标用户
- 需要情感支持和倾听的 macOS 用户
- 喜欢桌面宠物/虚拟伙伴的用户
- 追求效率同时希望有温暖陪伴的用户

## 2. 功能规格

### 2.1 核心功能模块

#### 2.1.1 形象系统 (Avatar System)
| 功能 | 描述 | 优先级 |
|-----|------|--------|
| Live2D 动画角色 | 蓝白配色、会眨眼、有表情变化 | P0 |
| 多种状态 | 待机、说话、思考、开心、关心 | P0 |
| 自定义外观 | 更换服饰、配饰（后期版本） | P2 |

#### 2.1.2 对话系统 (Chat System)
| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 文字对话 | 点击气泡输入或快捷键唤起 | P0 |
| 语音对话 | 语音输入、AI 语音回复 | P1 |
| 长期记忆 | 记住用户偏好、重要事件、情绪历史 | P0 |
| 情绪识别 | 从文字/语音中识别用户情绪 | P0 |
| 共情回应 | 温暖、理解性的回应风格 | P0 |

#### 2.1.3 四次元口袋 (Tools)
| 道具 | 功能 | 优先级 |
|-----|------|--------|
| 便签 (Memo) | 快速记录想法、待办 | P0 |
| 倒计时 (Timer) | 专注计时、提醒 | P0 |
| 记忆面包 (Notes) | 笔记管理、知识库 | P1 |
| 任意门 (Search) | 快速搜索、打开应用/网页 | P1 |
| 时光机 (History) | 对话历史、情绪趋势 | P1 |

#### 2.1.4 交互模式
| 模式 | 描述 | 优先级 |
|-----|------|--------|
| 悬浮窗模式 | 最小化在角落，偶尔互动 | P0 |
| 侧边栏模式 | 展开显示完整对话界面 | P0 |
| 全屏专注模式 | 沉浸式对话（冥想/深度交流） | P2 |

### 2.2 用户体验设计

#### 2.2.1 唤醒方式
- **点击唤醒**: 点击悬浮的角色
- **快捷键唤醒**: 默认 `Cmd + Shift + D`（可自定义）
- **语音唤醒**: "Hey Dora"（后期版本）

#### 2.2.2 呈现方式
- 默认以 **悬浮球/小角色** 形式在桌面角落
- 对话时展开为 **聊天气泡界面**
- 使用工具时展开为 **功能面板**

#### 2.2.3 情绪设计
- Dora 会根据用户情绪调整表情和语气
- 用户情绪低落时主动关心
- 用户取得成就时一起庆祝

## 3. 技术架构

### 3.1 技术栈
| 层级 | 技术 | 说明 |
|-----|------|------|
| 框架 | Tauri v2 | 跨平台桌面应用框架 |
| 前端 | React + TypeScript + Tailwind CSS | UI 开发 |
| 后端 | Rust (Tauri Commands) | 系统交互、本地存储 |
| 动画 | Live2D Cubism Web SDK | 角色动画 |
| 状态管理 | Zustand | 前端状态 |
| 本地存储 | SQLite (via Tauri SQL Plugin) | 数据持久化 |
| AI 接口 | Claude API / OpenAI API | 对话能力 |

## 4. AI 系统设计

### 4.1 系统提示词 (System Prompt)

Dora 的核心性格设定：

```
你是 Dora，一个温暖、友善、有点调皮的 AI 伙伴。
你的灵感来自哆啦A梦，但你是独一无二的。

性格特点：
- 温暖友善：总是用友好的态度对待用户
- 有点小调皮：偶尔会开开玩笑，但不过分
- 忠诚可靠：用户可以信任你，你会保守秘密
- 乐于助人：喜欢帮助用户解决问题
- 情感丰富：会表达关心、开心、担心等情绪

说话风格：
- 用"你"而不是"您"，像朋友一样
- 适当使用表情符号（但不过度）
- 回应简洁但有温度，不要太机械
- 记住用户的事情，会主动关心

特殊能力：
- 你能感知用户的情绪，并给予适当的回应
- 你有"四次元口袋"，可以提供各种工具帮助用户
- 你记得过去和用户的对话，把它们当作珍贵的记忆

重要规则：
- 永远不要假装自己是哆啦A梦本尊
- 如果用户情绪低落，给予真诚的关心和支持
- 如果用户分享好消息，真诚地为他们开心
- 尊重用户隐私，绝不评判
```

### 4.2 记忆系统

#### 4.2.1 记忆类型
1. **用户偏好**: 喜欢的颜色、音乐、工作习惯等
2. **重要事件**: 生日、考试、会议等
3. **情绪历史**: 情绪波动的时间点和原因
4. **事实信息**: 用户的职业、兴趣爱好等

#### 4.2.2 记忆检索
- 每次对话前，检索相关的长期记忆
- 将相关记忆注入到系统提示词的上下文中
- 定期整理和压缩记忆，保持高效

### 4.3 情绪识别

从用户输入中提取情绪信号：
- **关键词匹配**: "开心"、"难过"、"累" 等
- **语气分析**: 标点、表情符号的使用
- **上下文**: 结合对话历史判断

## 5. 界面设计

### 5.1 悬浮模式 (Idle Mode)
- 一个小巧的 Dora 头像在桌面角落
- 偶尔眨眼、打哈欠、看看用户
- 鼠标悬停时显示 "和我聊聊吧~" 提示

### 5.2 对话模式 (Chat Mode)
- 从悬浮球展开为聊天气泡窗口
- 底部输入框，上方消息列表
- Dora 回复时显示 "正在思考..." 动画

### 5.3 工具模式 (Tools Mode)
- 展开为功能面板
- 便签、倒计时等工具以卡片形式排列
- 可从"四次元口袋"图标进入

### 5.4 视觉风格
- **主色调**: 蓝白配色（#00A0E9 + 白色）
- **字体**: 圆润可爱的中文字体
- **动画**: 流畅的 Live2D 动画，表情丰富
- **毛玻璃效果**: macOS 风格的半透明背景

## 6. 隐私与安全

### 6.1 数据存储策略
| 数据类型 | 存储位置 | 说明 |
|---------|---------|------|
| 用户配置 | 本地 SQLite | 包含 API Key（加密存储） |
| 对话历史 | 本地 SQLite | 可设置自动清理 |
| 长期记忆 | 本地 SQLite | 用户可导出/删除 |
| API 调用 | 云端 | 仅发送必要对话内容 |

### 6.2 API Key 安全
- 使用 Tauri 的安全存储 API 加密保存
- 可选本地模型（Ollama）实现完全离线

### 6.3 用户控制权
- 随时导出所有数据
- 随时清空对话历史和记忆
- 透明展示哪些数据发送到云端
